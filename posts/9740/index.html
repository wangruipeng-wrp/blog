【Java并发】创建线程的方式

发表于 2020-06-19 | 分类于 并发

本文将介绍Java中创建线程的两种方式，以及两种方式的优缺点的比较，最后将会简单的看一下源码。

一、实现线程的两种方式

在 Oracle官方文档 中，明确说明了Java实现线程的方式就只有两种。一种是 实现 Runnable 接口，另一种是 继承 Thread 类

  1. 实现 Runnable 接口

  2. 继承 Thread 类

二、两种实现方式的比较

通常情况下我们需要去创建一个线程的时候会选择「实现 Runnable 接口」的方式，事实上 实现 Runnable 接口 的方式确实优于 继承 Thread 类 的方式。

  1. 从程序架构的角度考虑的话，创建线程的工作和线程要做的工作应该是分开的，也就是解耦的。实现 Runnable 接口 的方式就将线程要执行的代码也就是 run 方法里面的内容以及创建线程的机制也就是 Thread 类进行分离了，从而解耦。而 继承 Thread 类 的方式就没有做到这一点。

  2. 从执行效率的角度考虑的话，实现 Runnable 接口 的方式也是优于 继承 Thread 类 的方式的。如果使用 继承 Thread 类 的方式去创建线程每次新建一个任务时都要去创建一个新的线程，而创建线程的消耗是比较大的。实现 Runnable 接口 的方式可以让我们在后续利用一些线程池之类的工具，减少对资源的损耗。

  3. 从代码设计的角度去考虑的话，由于 Java 是不支持多继承的，实现接口的方式肯定是优于继承类的方式。

思考题：如果同时使用两种方式会怎么样

很明显，实现 Runnable 接口方式传入的 Runnable 对象被转换成了 target 对象，判空之后再执行 run 方法。但是我们又重写了 Thread 类的 run 方法，直接覆盖掉了 Thread 类中原有的 run 方法，所以 Runnable 对象的 run 方法并没有被调用，也就没有执行。

总结

  1. 实现线程的方式只有两种，分别是「实现 Runnable 接口」和「继承 Thread 类」
  2. 准确的讲，创建线程的方式只有一种那就是构造 Thread 类，而 Thread 类中提供了两种实现线程的执行单元的方式
    • 方式一：实现 Runnable 接口的 run 方法，并把 Runnable 实例传给 Thread 类（实现 Runnable）
    • 方式二：重写 Thread 的 run 方法（继承 Thread）
